🏥 About This Project

The Diabetes Prediction project is a comprehensive machine learning application designed to predict diabetes status using the Pima Indians Diabetes dataset. This project demonstrates best practices in data science, including modular code architecture, multiple model evaluation, and a user-friendly web interface.

📊 Dataset Information

Source

Pima Indians Diabetes Database from the National Institute of Diabetes and Digestive and Kidney Diseases

Size

768 samples with 8 clinical features and 1 binary target variable (Outcome)

Features

  • Pregnancies
  • Glucose
  • Blood Pressure
  • Skin Thickness
  • Insulin
  • BMI
  • Diabetes Pedigree Function
  • Age

🤖 Machine Learning Models

This project trains and compares 4 different machine learning algorithms for diabetes prediction:

Logistic Regression

A linear classification algorithm that models the probability of a binary outcome.

  • Fast and interpretable
  • Works well for linear relationships
  • Good baseline model

Random Forest

An ensemble method combining multiple decision trees for robust predictions.

  • Handles non-linear patterns
  • Less prone to overfitting
  • Feature importance ranking

Support Vector Machine

A powerful algorithm finding optimal decision boundaries in high-dimensional space.

  • Effective in high dimensions
  • Memory efficient
  • Versatile kernel options

K-Nearest Neighbors

A simple, instance-based algorithm that classifies based on nearby training examples.

  • Simple and intuitive
  • No training phase
  • Works with non-linear data

📈 Performance Metrics Explained

Accuracy

The proportion of correct predictions (both TP and TN) among all predictions.

(TP + TN) / (TP + TN + FP + FN)

Precision

Of positive predictions, how many were actually correct? Important to reduce false alarms.

TP / (TP + FP)

Recall (Sensitivity)

Of actual positive cases, how many did we catch? Critical for medical screening.

TP / (TP + FN)

F1 Score

Harmonic mean of precision and recall. Balances both metrics.

2 × (Precision × Recall) / (Precision + Recall)
Key Point: In medical screening, recall is often more important than precision because missing a positive case (false negative) is more costly than a false alarm (false positive).
